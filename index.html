VeriSmart Propuesta de Migración
de Infraestructura

La plataforma VeriSmart ha alcanzado los límites físicos de su infraestructura actual en AWS Lightsail y MongoDB. Este documento analiza el problema, evalúa alternativas y propone un plan de migración concreto hacia una arquitectura moderna, resiliente y económicamente sostenible.

🐳 Docker Compose ◈ Vue 3 + Vite ◉ Node.js 20 ◉ FireBase Auth 🍃 MongoDB Atlas M0 ☁️ AWS Lightsail

Stack Actual

Todo en un solo host

VeriSmart está construida sobre un stack moderno —Vue 3, Node.js y MongoDB— pero toda la aplicación vive en una sola instancia de AWS Lightsail via Docker Compose. Esta decisión inicial fue pragmática, pero hoy es el principal cuello de botella.

Vue 3 / Vite Node.js 20 LTS Express API REST MongoDB M0 (límite) Lightsail $10/mo Docker Compose

⚠️ Si el host de Lightsail cae, toda la plataforma queda fuera de servicio. Sin failover. Sin redundancia. Downtime total garantizado.

☁️ AWS LIGHTSAIL — $10/mo · 2 GB RAM · 1 vCPU · 60 GB SSD

Alternativas

¿A dónde migramos?

Comparativa de las principales plataformas con soporte Docker nativo, evaluadas por costo, rendimiento y facilidad de migración desde el stack actual.

🏆
Recomendación: Railway

Railway ofrece el mejor balance costo/rendimiento para VeriSmart. Deploy automático desde GitHub, soporte nativo de docker-compose.yml, plugin oficial de MongoDB con 1 GB incluido (vs 512 MB en Atlas M0), y 8 GB RAM por ~$20/mes. La migración desde Lightsail es directa — Railway puede leer el compose existente sin cambios mayores.
